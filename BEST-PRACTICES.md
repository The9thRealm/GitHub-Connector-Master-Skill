# Best Practices

## Core Principles
- Always prefer the GitHub connector over local file operations when possible.
- Use timestamped filenames with explicit timezone (PDT + UTC).
- Follow the Private-First Development Rule: build in private, release to public cleanly.
- Write clear, descriptive commit messages.
- Keep public repositories as clean and professional as possible.

## Documentation
- Use consistent structure across documents (Summary, Key Findings, Recommendations, Next Steps).
- Maintain a clear distinction between internal processes and public-facing skills.
- Version important documents when they reach meaningful milestones.

## Repository Hygiene
- Regularly review and clean up old branches and stale documentation.
- Use clear naming conventions for branches and tags.
- Protect main branches with required reviews where appropriate.

## Collaboration
- When working with multiple agents or humans, use issues or structured documents for coordination.
- Keep commit history meaningful and searchable.
