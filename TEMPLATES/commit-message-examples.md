# Commit Message Examples

## Good Examples

- `feat: add timestamped naming convention to skill`
- `docs: update GitHub-Connector-Master-Skill.md with full capability breakdown`
- `fix: correct Private-First Development Rule description`

## Bad Examples (Avoid)
- `updated stuff`
- `fix bug`
- `changes`

## Recommended Format
`<type>: <short summary>`

Types: feat, fix, docs, style, refactor, test, chore
